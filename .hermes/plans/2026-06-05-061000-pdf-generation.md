# Plan : Génération PDF réelle — AssoAI

**Date** : 2026-06-05
**Statut** : Planifié
**Kanban** : `assoai-pdf`

---

## 1. Objectif

Remplacer la génération PDF actuelle (HTML → `window.print()`) par une **vraie génération PDF binaire** avec :
- Un style professionnel unifié (marque **Imprimelle Côte d'Ivoire**) pour les 4 types de documents
- Inclusion du logo Imprimelle : `https://imgur.com/eSa965q`
- Toutes les infos du document (client, articles, prix, remise, etc.)
- Images produits si présentes dans les données
- Téléchargement direct (pas de boîte de dialogue d'impression)

---

## 2. Approche technique

### Architecture choisie : Puppeteer en microservice Docker

```
┌─────────┐    POST /api/pdf    ┌──────────────┐    HTML→PDF    ┌──────────┐
│  AssoAI │ ─────────────────→  │ nginx:alpine │ ─────────────→ │ pdf-svc  │
│  (SPA)  │ ←──── PDF blob ──── │  (reverse    │ ←── PDF ────── │ Puppeteer│
└─────────┘                     │   proxy)     │                └──────────┘
                                └──────────────┘
```

**Pourquoi Puppeteer plutôt que html2pdf.js ?**
- Qualité PDF native (texte sélectionnable, pas d'image aplatie)
- Rendu CSS fidèle (pas de limitations html2canvas)
- Support des images distantes (logo Imprimelle, images produits Supabase)
- Génération côté serveur = pas de popup bloquée, pas de boîte de dialogue d'impression

### Conteneur PDF Service

- **Image de base** : `node:22-alpine` + Chromium headless
- **Framework** : Express.js minimal (1 seul endpoint)
- **Port interne** : 3000
- **Endpoint** : `POST /api/pdf` — body HTML → réponse PDF binaire
- **Timeout** : 30s (suffisant pour du HTML simple + images)

## 5. Flux de génération PDF

```typescript
// pdfGenerator.ts — nouvelle fonction
async function generateRealPDF(templateType, data): Promise<Blob> {
  const html = buildHTML(templateType, data);  // nouveau builder
  const response = await fetch('/api/pdf', {
    method: 'POST',
    headers: { 'Content-Type': 'text/html' },
    body: html,
  });
  return await response.blob();
}
```

```typescript
// PDFActionMessage.tsx — handleDownloadPDF modifié
async function handleDownloadPDF() {
  const pdfBlob = await generateRealPDF(templateType, templateData);
  const url = URL.createObjectURL(pdfBlob);
  
  // Téléchargement direct
  const a = document.createElement('a');
  a.href = url;
  a.download = `${templateType}_${documentNumber}.pdf`;
  a.click();
}
```

## 6. Déploiement

### docker-compose.yml (extrait à ajouter)

```yaml
pdf-service:
  build: ./pdf-service
  restart: unless-stopped
  expose:
    - "3000"
  networks:
    - hermes-network

assoai:
  # ... existant ...
  volumes:
    - ./assoai-pdf.conf:/etc/nginx/conf.d/pdf.conf:ro  # ← ajout
```

### assoai-pdf.conf (nginx)

```nginx
location /api/pdf {
    proxy_pass http://pdf-service:3000/api/pdf;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    client_max_body_size 10M;  # HTML + images base64
}
```

### Commande de déploiement

```bash
# Sur le VPS
docker compose build pdf-service
docker compose up -d --force-recreate pdf-service assoai
```

## 7. Tests de validation

| Test | Critère |
|---|---|
| Facture → PDF | Téléchargement .pdf, logo présent, tableau correct, total en vert |
| Devis → PDF | Validité affichée, structure/méthode, pas d'erreur |
| Commande → PDF | Image produit visible si présente, date livraison |
| CDC → PDF | Multi-enseignes, équipe listée, sections matériaux |
| Image manquante | Fallback gracieux (pas de broken image) |
| Timeout | Erreur utilisateur propre si > 30s |
| Service down | Message d'erreur + fallback vers window.print() |

## 9. Risques et mitigations

| Risque | Mitigation |
|---|---|
| Puppeteer ne trouve pas Chromium dans Alpine | Utiliser `chromium` au lieu de `chrome` dans le launcher Puppeteer |
| Images distantes (logo imgur) lentes | Timeout 30s + fallback sans image |
| Service PDF down | Fallback vers l'ancien `window.print()` |
| Fichiers volumineux (CDC multi-enseignes) | `client_max_body_size 10M` dans nginx |
